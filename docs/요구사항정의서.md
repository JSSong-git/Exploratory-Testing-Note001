## 1. 개요

### 1.1 목적

본 문서는 **Exploratory Testing Extension v2.0.0** 이 충족해야 하는 기능·데이터·인터페이스·비기능 요구사항을 빠짐없이 정의한다. 탐색적 테스트(Exploratory Testing) 세션 중 발견 사항을 기록·캡처·검토·보고·내보내기할 수 있는 Chrome 확장 프로그램의 구현 기준이다.

### 1.2 제품 개요

| 항목 | 내용 |
|------|------|
| 패키지명 | `exploratory-testing-extension` |
| 표시명 | Exploratory Testing Extension |
| 사이드 패널 제목 | Test Session |
| 부제 | Exploratory Testing |
| 액션 툴팁 | Exploratory Testing |
| 아이콘 | `icon/128.png` (128×128) |

### 1.3 주요 가치

- 세션 단위로 결함·관찰·개선·질문을 구조화하여 기록한다.
- 전체/영역 스크린샷과 주석(화살표, 사각형, 텍스트, 가림)을 지원한다.
- 저장 전 검토, 세션 보고서, Markdown/JSON/CSV/HTML 내보내기·JSON 가져오기를 제공한다.
- 세션 메타데이터와 스크린샷 바이너리를 분리 저장하여 스토리지 쿼터 위험을 줄인다.

### 1.4 이해관계자

| 역할 | 관심사 |
|------|--------|
| QA / 테스터 | 세션 중 빠른 기록·캡처·보고서 |
| 개발자 | 빌드·테스트·확장 로드 |
| 리뷰어 | 요구사항 대비 구현·회귀 검증 |

### 1.5 범위

**포함:** 사이드 패널, 백그라운드 서비스 워커, 콘텐츠 스크립트, 어노테이션 에디터, 저장 확인 UI, 세션 보고서, 내보내기/가져오기, 알림, 로컬 저장소.

**제외:** 서버 동기화, 다중 사용자 협업, 브라우저 외 플랫폼, 한국어 UI(제품 UI는 영어 고정).

## 21. 용어집

| 용어 | 정의 |
|------|------|
| Session | 한 번의 탐색적 테스트 세션 데이터 집합 |

| Question | 질문 (내부 키 `question`) |

| Area capture | 사용자 선택 영역 캡처 + 주석 |
| Capturable tab | `http://` 또는 `https://` URL을 가진 탭 |
| imageId | IndexedDB에 저장된 스크린샷 식별자 |
| Compose draft | Record 탭 작성 중 자동 저장 초안 |
| Broadcast | `SESSION_CHANGED`로 UI에 세션 변경 통지 |
