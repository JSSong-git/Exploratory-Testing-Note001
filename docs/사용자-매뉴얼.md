# Exploratory Testing Extension 사용자 매뉴얼

**버전:** 2.0.0  
**대상:** Chrome / Chromium 기반 브라우저  
**UI 언어:** 영어 (본 매뉴얼은 한글 설명)

탐색적 테스트(Exploratory Testing) 세션 중 발견한 결함·관찰·개선안·질문을 기록하고, 스크린샷과 함께 보고서를 만들 수 있는 Chrome 확장 프로그램입니다.

## 1. 개요

### 무엇을 할 수 있나요?

| 기능 | 설명 |
|------|------|
| 세션 기록 | 결함, 관찰, 개선안, 질문을 제목·설명(Markdown)·URL과 함께 저장 |
| 스크린샷 | 전체 화면 또는 선택 영역 캡처 후 화살표·사각형·텍스트·가림 처리 |
| 검토 후 저장 | 저장 전 내용을 한 번 더 확인 |
| 보고서 | 유형별 차트와 목록이 있는 세션 리포트 미리보기 |
| 내보내기 | Markdown, JSON, CSV, HTML 등으로 다운로드 |
| 가져오기 | 이전에 내보낸 JSON 세션을 다시 불러오기 |

### 권장 사용 흐름

1. 테스트할 웹 페이지를 연다.  
2. 확장 프로그램 사이드 패널을 연다.  
3. 발견 사항을 기록하거나 캡처한다.  
4. 필요 시 보고서를 확인하고 내보낸다.

## 2. 설치 및 실행

### 2.1 개발자용 로드 (소스에서 빌드)

1. 프로젝트 폴더에서 다음을 실행합니다.

```bash
npm install
npm run build
```

2. Chrome 주소창에 `chrome://extensions` 를 입력합니다.  
3. **개발자 모드**를 켭니다.  
4. **압축해제된 확장 프로그램을 로드합니다**를 클릭합니다.  
5. 다음 폴더를 선택합니다.

```text
.output/chrome-mv3
```

> **주의:** `.output/chrome-mv3-dev` 는 개발 서버(`npm run dev`)가 켜져 있을 때만 사용하세요.  
> 일반 사용·검증은 **`.output/chrome-mv3`** 를 권장합니다.

### 2.2 사이드 패널 열기

- 툴바의 확장 프로그램 아이콘을 클릭하면 **사이드 패널**이 열립니다.  
- 패널 제목: **Test Session** / 부제: **Exploratory Testing**

### 2.3 코드 업데이트 후

1. `npm run build` 를 다시 실행합니다.  
2. `chrome://extensions` 에서 해당 확장의 **새로고침(↻)** 을 누릅니다.  
3. 테스트 중이던 웹 페이지도 **새로고침**하는 것이 안전합니다.

## 3. 화면 구성

사이드 패널 상단 탭:

| 영문 UI | 의미 |
|---------|------|
| **Record** | 새 기록 작성 / 캡처 |
| **Saved** | 저장된 기록 목록 |

하단에 **More** 메뉴가 있으면 다음을 사용할 수 있습니다.

- **Preview report** — 세션 보고서 열기  
- **Export** — 내보내기  
- **Import** — JSON 가져오기  
- **Reset session** — 세션 초기화  

---

## 4. 기록 유형

기록 유형은 ISTQB·탐색적 테스트에서 쓰는 영문 라벨로 표시됩니다.

| UI 라벨 | 내부 키 | 설명 |
|---------|---------|------|
| **Defect** | bug | 결함 (실패를 유발할 수 있는 결점) |
| **Observation** | note | 관찰 기록 |
| **Improvement** | idea | 개선 제안 |
| **Question** | question | 질문 / 확인 필요 사항 |

유형 탭을 눌러 전환한 뒤 제목과 설명을 작성합니다.

---

## 5. 기본 기록 작성

스크린샷 없이 텍스트만 저장하는 방법입니다.

1. **Record** 탭을 선택합니다.  
2. 유형(Defect / Observation / Improvement / Question)을 고릅니다.  
3. **Title (required)** 에 제목을 입력합니다. *(필수)*  
4. 설명란에 Markdown으로 상세 내용을 작성합니다.  
   - 예: 재현 절차, 기대 결과, 실제 결과  
   - **Edit** / **Preview** 로 편집·미리보기를 전환할 수 있습니다.  
5. **Review** 를 누릅니다.  
6. 검토 화면에서 내용을 확인한 뒤 **Confirm save** 로 저장합니다.  
7. **Saved** 탭에서 저장된 항목을 확인합니다.

### Markdown 작성 팁

| 입력 | 결과 |
|------|------|
| `**굵게**` | **굵게** |
| `- 목록` | 글머리 목록 |
| `` `코드` `` | 인라인 코드 |
| `## 제목` | 소제목 |

## 6. 전체 화면 캡처 (Full capture)

현재 보고 있는 웹 페이지의 **보이는 영역 전체**를 캡처합니다.

1. 캡처할 `http://` 또는 `https://` 페이지를 엽니다.  
2. 사이드 패널 **Record** 탭에서 **Full capture** 를 클릭합니다.  
3. 미리보기와 함께 저장 확인 창이 열립니다.  
4. 유형·제목·설명을 입력하거나 수정합니다.  
5. **Save** 로 저장합니다.

> 제목이 비어 있으면 저장 버튼이 비활성화됩니다.

---

## 7. 영역 캡처 (Area capture)

페이지의 **일부 영역만** 선택해 캡처하고, 화살표·메모 등을 추가한 뒤 저장합니다.

### 7.1 순서

1. 테스트할 `http://` / `https://` 페이지를 엽니다.  
2. 사이드 패널 **Record** 에서 **제목(Title)을 먼저 입력**합니다. *(필수)*  
3. **Area capture** 를 클릭합니다.  
4. 웹 페이지로 전환되면 화면이 약간 어두워지고 안내가 표시됩니다.  
5. **마우스로 드래그**해 영역을 선택합니다.  
   - 취소: **Esc**  
6. 주석 편집기(Annotation Editor)가 열립니다.  
7. 도구로 표시를 추가한 뒤 **Save** 를 누릅니다.  
8. 저장 확인 창에서 제목·설명을 확인·수정하고 **Save** 합니다.  
9. 페이지에 저장 안내가 뜨고, 사이드 패널 **Saved** 에 항목이 나타납니다.

### 7.2 주석 편집기 도구

| 도구 | 설명 |
|------|------|
| **Arrow** | 화살표 |
| **Rectangle** | 사각형 |
| **Text** | 텍스트 라벨 (클릭 후 입력) |
| **Redact** | 가림(블러) 처리 |
| **Undo** | 실행 취소 (Ctrl+Z) |
| **Save** | 편집 완료 |
| **Cancel** | 편집 취소 |

단축키:

- **Esc** — 취소  
- **Ctrl+Z** — 실행 취소  
- **Ctrl+Enter** — 저장(편집 완료)

### 7.3 Area capture가 안 될 때

- 제목을 입력했는지 확인합니다.  
- `chrome://`, 확장 프로그램 페이지, Chrome 웹스토어 등에서는 캡처할 수 없습니다.  
- 확장 프로그램을 새로고침한 뒤에는 **웹 페이지도 새로고침**하세요.  
- 반드시 **웹 페이지 위**에서 드래그해야 합니다. (사이드 패널 위가 아님)

---

## 8. 저장된 기록 관리

1. **Saved** 탭을 엽니다.  
2. 목록에서 항목을 클릭하면 상세(제목, 설명, URL, 스크린샷, 시각)를 볼 수 있습니다.  
3. **Edit** — 제목·설명을 수정한 뒤 **Update**  
4. **Delete** — 삭제 확인 후 제거  

스크린샷이 있는 항목은 상세 화면에서 이미지를 확인할 수 있습니다.

---

## 9. 세션 보고서

1. 사이드 패널에서 **Preview report** (또는 More 메뉴)를 선택합니다.  
2. 새 탭에 **Session Report** 가 열립니다.  
3. 확인할 수 있는 내용:
   - 세션 시작 시각  
   - 유형별 건수 차트  
   - 제목·설명·URL·스크린샷·기록 시각 표  
4. 유형 필터와 검색창으로 목록을 좁힐 수 있습니다.

## 10. 내보내기 / 가져오기

### 10.1 내보내기 (Export)

기록이 1건 이상일 때 **Export** 메뉴를 사용합니다.

| 형식 | 설명 |
|------|------|
| **Markdown (.zip)** | Markdown + 이미지 폴더 (기본 권장) |
| **Markdown inline (.md)** | 이미지가 인라인으로 포함된 단일 MD |
| **JSON** | 세션 데이터 (다시 가져오기 가능) |
| **CSV** | 표 형태 |
| **Standalone HTML** | 단독 HTML 보고서 |

파일은 브라우저 다운로드 폴더에 저장됩니다.

### 10.2 가져오기 (Import)

1. **Import** 를 선택합니다.  
2. 이전에 내보낸 **JSON** 파일을 고릅니다.  
3. 세션이 교체·복원됩니다.

> 같은 확장에서 내보낸 JSON과, 구버전(레거시) JSON 형식도 지원합니다.

---

## 11. 세션 초기화

**Reset session** 을 선택하면 현재 세션의 **모든 기록과 관련 스크린샷**이 삭제됩니다.

- 확인 대화상자가 표시됩니다.  
- 되돌릴 수 없으므로, 필요하면 먼저 **Export** 하세요.

---

## 12. 주의사항

1. **캡처 가능 페이지**  
   - `http://`, `https://` 일반 웹 페이지만 가능합니다.  
   - `chrome://`, `edge://`, PDF 뷰어, 일부 특수 페이지는 불가합니다.

2. **사이드 패널과 웹 탭**  
   - Area capture는 웹 페이지에서 드래그해야 합니다.  
   - Full capture는 캡처 가능한 웹 탭을 대상으로 합니다.

3. **데이터 저장 위치**  
   - 세션 메타데이터: Chrome 로컬 스토리지  
   - 스크린샷: IndexedDB (세션 JSON에 base64를 넣지 않음)

4. **UI 언어**  
   - 프로그램 화면은 영어입니다.  
   - Defect / Observation 등은 ISTQB·탐색적 테스트 용어를 따릅니다.

## 13. 문제 해결

| 증상 | 확인 / 조치 |
|------|-------------|
| 확장 오류에 `ws://localhost:3000` | 개발용(`chrome-mv3-dev`)을 로드한 상태입니다. `.output/chrome-mv3` 로 다시 로드하세요. |
| Area capture 클릭 후 반응 없음 | 제목 입력 여부, http(s) 페이지 여부, 페이지 새로고침 후 재시도 |
| 드래그해도 편집기가 안 열림 | 영역이 너무 작지 않은지(최소 약 4×4px), 올바른 웹 탭인지 확인 |
| 저장했는데 Saved에 안 보임 | 확장 새로고침 후 최신 빌드(`.output/chrome-mv3`)인지 확인. 저장 성공 시 페이지에 안내 토스트가 뜹니다. |
| Full capture 실패 | 캡처할 웹 탭이 열려 있는지 확인 |
| 가져오기 실패 | JSON이 이 확장에서 내보낸 형식인지 확인 |
| 이미지가 “unavailable” | 세션만 남고 이미지 DB가 지워진 경우일 수 있습니다. 다시 캡처하거나 백업 JSON을 확인하세요. |
